SmartInvesting API Tài liệu độc lập

Trang tổng quan use case, cơ sở dữ liệu và luồng xử lý — chỉ để đọc, không phụ thuộc host hay build của backend. Chi tiết endpoint có thể đối chiếu OpenAPI/Swagger của dịch vụ khi chạy cục bộ.

1. Tổng quan

API phục vụ giả lập đầu tư (ví ảo, lệnh mua/bán, danh mục, giá thị trường) và quản lý chi tiêu (giao dịch thu/chi, ngân sách theo tháng, tổng quan dashboard). Xác thực bằng JWT; dữ liệu nghiệp vụ tách database với ASP.NET Identity (user/role).

3. Cơ sở dữ liệu

Hai SQL Server (connection riêng trong appsettings): AppDbcontext (nghiệp vụ) và AppIdentityDbcontext (AspNetUsers, roles, v.v.).

3.1. Các thực thể chính (AppDbcontext)

EntityVai tròLiên kết chính
Wallet Ví theo user; số dư Balance. UserId (khớp claim JWT, không FK sang Identity bắt buộc)
Category Hạng mục Income/Expense. Transaction, Budget
Transaction Giao dịch tiền mặt (note, amount, ngày). WalletId, CategoryId, optional AssetId
Budget Hạn mức chi theo tháng/năm. UserId, CategoryId; unique (UserId, CategoryId, Month, Year)
Asset Mã chứng khoán/quỹ; AssetType, CurrentPrice. Portfolio, AssetPricesHistory
AssetPricesHistory Lịch sử giá (chuẩn bị cho biểu đồ; có thể chưa expose HTTP). AssetId
Portfolio Vị thế user trên một Asset: khối lượng, giá vốn TB. UserId, AssetId
InvestmentOrder Lệnh mua/bán; fee; RealizedProfitLoss khi bán. PortfolioId, WalletId
IncomeEvent Sự kiện thu (ví dụ cổ tức). WalletId, optional AssetId

4. Workflow & luồng API

4.1. Đăng nhập và gọi API

sequenceDiagram participant C as Client participant A as AuthAPI participant X as JWTControllers C->>A: POST login A-->>C: access_token C->>X: GET or POST with Bearer header X-->>C: JSON response

4.2. Giả lập đầu tư (mua)

  1. Đảm bảo đã có Asset trong DB (seed hoặc POST assets).
  2. Tạo / chọn Wallet đủ số dư.
  3. POST /api/investment-orders với OrderType = Buy: service trừ tiền ví, tạo/cập nhật Portfolio, ghi InvestmentOrder.
  4. Xem vị thế: GET /api/holdings/types/{typeId} với refreshMarketPrice=true nếu cần giá mới.

4.3. Chi tiêu và ngân sách

  1. POST /api/wallets/{id}/transactions với category Expense/Income — số dư ví thay đổi theo loại category.
  2. Tạo Budget cho (user, category, tháng, năm) — tổng đã chi trong summary/dashboard chỉ tính giao dịch Expense trên ví của user đó.
  3. GET /api/dashboard gom ví, NAV (theo giá hiện tại trong DB hoặc làm mới giá), chi tháng, các dòng budget.

4.4. Giá thị trường (cấu hình)

Simulation:UseLocalPrices = true → dùng Asset.CurrentPrice cho mọi loại (giả lập thuần DB). Ngược lại → FireAnt cho cổ phiếu/quỹ/ETF, fallback DB cho vàng.
