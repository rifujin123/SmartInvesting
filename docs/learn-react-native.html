Cách dùng trang này để tự làm cho riêng bạn — không chỉ đọc

Tài liệu chỉ “đúng” khi bạn tạo được một đoạn UI hoặc hành vi trên Expo mang dấu ấn của bạn (đặt tên component khác, chỉnh sốDuration, chỉnh màu). Dưới đây là quy trình lặp lại được cho mọi mục trong trang.

Chu trình 4 bước (mỗi lần học một ý nhỏ)

  1. Chọn “một mục tiêu nhìn thấy” — ví dụ: “ô chữ fade-in khi vào màn”, không phải “đọc hết Animated”.
  2. Vào dự án Expo — một file một screen (app/(tabs)/index.tsx hay screens/Lab.tsx tuỳ cấu trúc của bạn).
  3. Chép tối thiểu từ ví dụ trong trang, sửa một thứ của riêng bạn ngay trong lần đầu: tên prop, copy tiếng Việt, đổi duration.
  4. Tự chứng minh: điện thoại reload được; nếu lỗi đỏ — đọc chữ lỗi, sửa đến khi hết (hoặc ghi lại câu lỗi để tra docs).

Quy tắc “của riêng mình”

  • Đổi tên component / file sao cho bạn nhớ đó là bài bạn đã làm — ví dụ MyFadeCard, không để mãi Panel như sách.
  • Sau khi chạy được, chỉnh hai thông số chủ đề (timing, friction, delay, gap…) và quan sát — đó mới là cách “đồ” animation/layout.
  • Viết vào README hoặc ghi chú 1 dòng: “Hôm nay: fade-in 320ms → 520ms nhìn nặng tay hơn”.

Liên kết các mục trong trang này với việc bạn tay gõ

  • Hooks demo trên browser → tái hiện bằng useState trong một screen Expo.
  • Flex demo → chỉnh thật justifyContent / gap trong View.
  • Animation → làm đúng các Lab trong mục đó trên Expo (không bỏ qua phần “xong khi…”).
  • API → gọi một URL mock (vd. JSONPlaceholder), log response ra console rồi mới ghép vào UI.
Nếu bạn chỉ đọc không gõ — kiến thức sẽ trôi. Chuẩn tối thiểu: mỗi phiên học 20–40 phút = một commit nhỏ (hoặc một file đổi được thấy rõ trên máy).

React Native là gì?

React Native cho phép bạn viết ứng dụng mobile bằng JavaScript và component kiểu React. Khác với web: không dùng div, span, p — mà dùng View, Text, v.v. Giao diện được “dịch” sang native view trên iOS/Android (hoặc qua engine tuỳ stack bạn chọn).

Expo là bộ công cụ phổ biến phía trên RN: dễ cài, chạy nhanh trên điện thoại, tích hợp nhiều API thiết bị. Hầu hết người mới nên bắt đầu với Expo (SDK mới).

JavaScript cần thiết trước khi vào React

Bạn không cần “thành thạo tất cả”, nhưng nên quen các ý sau:

  • const / let, hàm mũi tên () => {}
  • Xâu và template literal `Chi ${name}`
  • Hủy cấu trúc object/array, spread ...
  • Gọi bất đồng bộ: async/await, fetch (khi học API)
  • Modules: import / export
// ví dụ nhanh ES module + async
import { Platform } from "react-native";

export async function loadJson(url) {
  const res = await fetch(url);
  if (!res.ok) throw new Error(`HTTP ${res.status}`);
  return res.json();
}

export const greeting = name => `Xin chào, ${name}`;

React căn bản (đủ để viết RN)

Component & JSX

Component là hàm trả về “cây JSX” giống HTML nhưng thực chất là lời gọi createElement.

function Welcome(props) {
  return <Text>Xin chào {props.name}</Text>;
}

Hooks: state & effect

  • useState: lưu trạng thái trong component (render lại khi đổi).
  • useEffect: chạy khi mount/update — gọi API, đăng ký listener, cleanup.
import { useEffect, useState } from "react";
import { Text, View } from "react-native";

export default function Clock() {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    const id = setInterval(() => setSeconds((s) => s + 1), 1000);
    return () => clearInterval(id);
  }, []);

  return (
    <View>
      <Text>Đã chạy: {seconds}s</Text>
    </View>
  );
}

Truyền props trong React Native (mở rộng)

Ngoài việc truyền từng tên cố định (name, title, tintColor…), bạn sẽ gặp thêm các mẫu sau khi ráp UI thật. Tất cả đều là “đưa dữ liệu hoặc hành vi từ cha xuống con” qua object props.

1. Giải nén và giá trị mặc định

Viết gọn hơn props.xyz; khi ghép vào JSX, các giá trị mặc định trong destructure giúp tránh undefined lộ liễu khi cha không truyền đủ.

function PriceTag({ symbol = "VN30", pct = 0 }) {
  return (
    <Text>
      {symbol} {pct >= 0 ? "+" : ""}
      {pct}%
    </Text>
  );
}

2. children — nội dung nằm giữa thẻ

Mọi thứ bọc trong <Card>...</Card> được truyền xuống qua prop đặc biệt children (hay destructure trong hàm).

import { StyleSheet, Text, View } from "react-native";

function ScreenCard({ title, children }) {
  return (
    <View style={styles.card}>
      <Text style={styles.title}>{title}</Text>
      {children}
    </View>
  );
}

/* Dùng: */
<ScreenCard title="Tổng quan">
  <Text>Nội dung bên trong chính là children</Text>
</ScreenCard>

const styles = StyleSheet.create({
  card: { padding: 16, borderRadius: 14, gap: 8 },
  title: { fontSize: 18, fontWeight: "700" },
});

3. Spread ...props / ...rest — chuyển tiếp xuống native component

Wrapper nút chỉ ép thêm kiểu, còn mọi sự kiện / style của thư viện vẫn truyền xuyên suốt bằng spread. 

import { Pressable, StyleSheet, Text } from "react-native";

const styles = StyleSheet.create({
  btn: { paddingVertical: 12, paddingHorizontal: 20, borderRadius: 14, backgroundColor: "#22d3ee" },
  dim: { opacity: 0.5 },
  txt: { fontWeight: "700", textAlign: "center" },
});

/** Chuyển onPress, style, accessibility... xuống Pressable qua {...rest} */
function PrimaryButton({ label, ...rest }) {
  return (
    <Pressable style={[styles.btn, rest.disabled && styles.dim]} {...rest}>
      <Text style={styles.txt}>{label}</Text>
    </Pressable>
  );
}

4. Props là hàm (callback từ con lên cha)

Cha giữ state; con nhận hàm để báo lại — onChangeText, onPress là các prop kiểu này.

import { Text, TextInput, View } from "react-native";

function LabeledInput({ label, value, onChangeText, placeholder }) {
  return (
    <View>
      <Text style={{ marginBottom: 4 }}>{label}</Text>
      <TextInput
        value={value}
        onChangeText={onChangeText}
        placeholder={placeholder ?? ""}
        style={{ borderWidth: 1, borderRadius: 8, padding: 10 }}
      />
    </View>
  );
}

/* Cha: const [bio, setBio] = useState(""); ... */
<LabeledInput
  label="Ghi chú"
  value={bio}
  onChangeText={setBio}
  placeholder="Nhập text..."
/>

5. FlatList: prop renderItem (nhận một hàm)

Hàm này cũng là “prop” — bạn báo cho list cách render từng phần tử; các prop phụ thường gặp: keyExtractor, ItemSeparatorComponent, ListHeaderComponent.

import { FlatList, Pressable, Text, View } from "react-native";

function WalletPicker({ wallets, selectedId, onSelect }) {
  return (
    <FlatList
      data={wallets}
      keyExtractor={(w) => String(w.id)}
      renderItem={({ item }) => (
        <Pressable onPress={() => onSelect(item.id)}>
          <View style={{ opacity: selectedId === item.id ? 1 : 0.6 }}>
            <Text>{item.name}</Text>
          </View>
        </Pressable>
      )}
    />
  );
}

6. key trong danh sách (quan trọng, không nhầm với props thường)

Khi map tay: mỗi phần tử cần <View key={id}> ổn định. FlatList dùng keyExtractor thay vai trò đó. Không chọn key theo chỉ mục mảng nếu có thể tránh (i khi reorder sẽ lộ bug).

Khi không muốn “rơi tiếng” props qua mọi tầng

Nếu cùng một giá trị (theme, user đăng nhập, ngôn ngữ…) phải bơm qua 5–6 cấp component, có thể dùng Context API của React để các lá sâu đọc trực tiếp, thay cho prop drilling. Không lạm dụng: luồng dữ liệu rõ ràng vẫn ưu tiên props và state gần nơi dùng.

Hooks — minh họa trực quan (chạy trong trình duyệt)

React Native dùng cùng bộ hook của React. Các khối dưới đây là JavaScript thuần trên trang web nhưng mô phỏng đúng hành vi: khi “state” đổi thì phần hiển thị cập nhật; khi tắt effect thì có bước cleanup (giống return trong useEffect).

useState — đếm và re-render

Trong RN: const [n, setN] = useState(0). Mỗi lần gọi setN, component chạy lại (re-render) và UI đọc giá trị n mới — giống nút dưới đây cập nhật số và bộ đếm “lần vẽ lại”.

Đếm: 0

Số lần cập nhật (≈ render): 0

useState — bật/tắt một phần UI

Rất phổ biến: bool state cho menu, bottom sheet, “xem thêm”. Thử bật/tắt khối dưới — trên RN bạn vẫn dùng useState(true/false) rồi điều kiện trong JSX.

Khi tắt checkbox, panel này ẩn — tương đương {visible && <View>...</View>}.

useState — “controlled input” (TextInput)

Ô nhập dưới đây điều khiển dòng chữ bên dưới: state = nội dung ô. Trên RN bạn dùng <TextInput value={name} onChangeText={setName} />.

useEffect — side effect có cleanup (đồng hồ)

Bật công tắc = effect chạy: đăng ký setInterval. Tắt = chạy cleanup (clearInterval) và đồng hồ dừng hẳn. Mảng phụ thuộc [] trong ví dụ đồng hồ nghĩa là chỉ khởi tạo khi vào màn; trong app thực có thể thêm [userId] để sync lại khi đổi user.

Tick: 0

// Tóm ý RN — các demo trên tương ứng
const [count, setCount] = useState(0);
const [on, setOn] = useState(true);
useEffect(() => {
  if (!on) return;
  const id = setInterval(() => setTick((t) => t + 1), 1000);
  return () => clearInterval(id);
}, [on]);

React Native: View, Text, và bố cục

  • View: khối bố cục (như một div trừu tượng).
  • Text: mọi chữ phải nằm trong Text (không gõ chữ “rơi tự do” vào View).
  • Pressable/TouchableOpacity: vùng bấm (nút).
  • ScrollView: cuộn nội dung ngắn.
  • FlatList/SectionList: danh sách dài (virtualization).
  • Image: hiển thị ảnh (đường dẫn từ mạng cần kích thước).
import { View, Text, StyleSheet } from "react-native";

export function Card({ title }) {
  return (
    <View style={styles.card}>
      <Text style={styles.title}>{title}</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  card: {
    padding: 16,
    borderRadius: 16,
    backgroundColor: "#18181b",
    borderWidth: 1,
    borderColor: "#3f3f46",
  },
  title: { color: "#fafafa", fontSize: 16, fontWeight: "700" },
});
Mặc định trong RN: flexDirection là 'column', ngược với CSS web là row. Hãy coi mọi màn là “Flexbox chiến” — chỉ có một số thuộc tính so với web (ví dụ không có shorthand border đầy đủ một dòng trong mọi phiên bản SDK).

Style trong React Native và so với CSS

  • StyleSheet.create: gom kiểu, tối ưu, tránh tạo object mới vô cớ mỗi render.
  • Không có cascade như CSS chọn selector — style áp vào đúng node qua props style.
  • Đơn vị: số không gắn px; coi là “density-independent pixels” giống ý của dp.
  • Đổ bóng: iOS là shadow*; Android là elevation.
  • Khi học RN, hãy tập nghĩ bằng hệ trục trục dọc + flex trước khi chồng absolute.
// Tab bar trong RN — ý chính: row, space-around, chiều cao + SafeAreaInsets
const styles = StyleSheet.create({
  tabBar: {
    flexDirection: "row",
    justifyContent: "space-around",
    alignItems: "center",
    paddingHorizontal: 8,
    paddingBottom: Platform.select({ ios: 12, android: 8 }),
    borderTopWidth: StyleSheet.hairlineWidth,
    borderTopColor: "#27272a",
    backgroundColor: "rgba(16,16,18,0.9)",
  },
});

Style phổ biến trong thiết kế hiện đại — làm gì, khi nào dùng

Bảng dưới gắn thuộc tính với ngữ cảnh thực tế app (dashboard, danh sách, form). Ấn các nút để thấy khác biệt trực quan. Trên React Native dùng tên CamelCase trong object style (ví dụ justifyContent: "space-between").

justifyContent (trục chính) — chia khoảng trống của hàng / cột

Phổ biến: space-between (hai đầu thanh toolbar), center ( căn giữa nhóm nút ), flex-start / flex-end. Thanh tab thường là row + space-around hoặc chia đều ô.

justify-content: flex-start

alignItems (trục phụ) — căn các con trên chiều vuông góc trục chính

Cột có chiều cao cố định + vài nút/thanh: chọn để các phần tử ôm trái, giữa, phải, hoặc kéo dài full bề ngang (stretch — mặc định theo chiều ngang của cột).

gap — khoảng cách đều giữa các con

padding vs margin

Padding: khoảng “lót” bên trong khung (nội dung không dính viền) — ô input, card, nút. Margin: khoảng “đẩy” ra ngoài so với hàng xóm — tránh chồng card, chừa chỗ tay cầm.

Animation cơ bản trong React Native

RN tích hợp API Animated (built-in): bạn có các giá trị động rồi “chạy” chúng theo thời gian (timing), lò xo (spring), hoặc chậm dần (decay). Mục tiêu thực dụng: mở màn nhẹ, nút bấm co giãn, toast / badge — ít fps hơn là quả animation dài không cần thiết.

Phần dưới tách hai lớp: A) Lab — bạn tự làm trên Expo theo checklist → B) Tra cứu (mục 1–10) khi quên chi tiết syntax. Trước khi đọc sang B, cố làm xong ít nhất Lab 1. Xem thêm phương pháp chung tại Cách tự thực hành.

Mục tiêu nhìn thấy: khối có nền tối và dòng chữ mờ dần thành rõ khi màn vừa hiện.

B) Tra cứu — chi tiết API

Các đoạn 1–10 bên dưới là tài liệu tham chiếu. Khi quên cụ thể (tham số spring, cú pháp parallel…), nhảy tới đúng mục; khi học lần đầu — ưu tiên làm Lab ở phần A trước.

1. Animated.Value và component function

Giữ value trong ref để không tạo mới mỗi render (tránh animation reset).

import { useEffect, useRef } from "react";
import { Animated, StyleSheet } from "react-native";

function Panel() {
  const opacity = useRef(new Animated.Value(0)).current;

  useEffect(() => {
    Animated.timing(opacity, {
      toValue: 1,
      duration: 320,
      useNativeDriver: true,
    }).start();
  }, [opacity]);

  return <Animated.View style={[styles.box, { opacity }]} />;
}

const styles = StyleSheet.create({
  box: { height: 48, margin: 16, borderRadius: 12, backgroundColor: "#27272a" },
});

2. useNativeDriver: true là gì?

Khi là true, phép biến đổi chạy trên luồng native (ưu tiên các thuộc tính được hỗ trợ như opacity, transform: translate, scale, rotate). Vuông góc là mượt, ít giật JS.

Giới hạn: với useNativeDriver: true bạn không animate height, width, backgroundColor trực tiếp trên cùng value kiểu cũ — phải tắt native driver (false) hoặc đổi cách tiếp cận (layout animation, hoặc Reanimated 2+/3 layout).

3. Animated.timing — tuyến tính hoặc easing

Animated.timing(translateY, {
  toValue: 0,
  duration: 260,
  useNativeDriver: true,
}).start(() => console.log("xong"));

Muốn cảm giác “ease out” có thể dùng Easing.out(Easing.cubic) (import Easing từ react-native).

4. Animated.spring — nút nhảy tự nhiên

Animated.spring(scale, {
  toValue: 1,
  friction: 4,
  useNativeDriver: true,
}).start();

5. translateX / scale trên một Animated.View

const anim = useRef(new Animated.Value(1)).current;

<Animated.View
  style={{
    transform: [{ scale: anim }],
  }}
>

Nhiều transform: một mảng, ví dụ [{ translateY }, { scale }] — có thể dùng nhiều Value khác nhau.

6. Ghép gesture nhấn: co rồi về (micro-interaction)

import { Pressable, Animated, Text } from "react-native";
import { useRef } from "react";

export function PulseButton({ onPress, label }) {
  const scale = useRef(new Animated.Value(1)).current;

  const down = () => {
    Animated.spring(scale, { toValue: 0.94, friction: 5, useNativeDriver: true }).start();
  };
  const up = () => {
    Animated.spring(scale, { toValue: 1, friction: 5, useNativeDriver: true }).start();
  };

  return (
    <Pressable onPress={onPress} onPressIn={down} onPressOut={up}>
      <Animated.View style={{ transform: [{ scale }], padding: 14, borderRadius: 14, backgroundColor: "#22d3ee" }}>
        <Text style={{ fontWeight: "800" }}>{label}</Text>
      </Animated.View>
    </Pressable>
  );
}

Text vẫn bọc trong Text — ví dụ rút gọn; thực tế có thể tách StyleSheet.

7. interpolate — một value, nhiều output

Ví dụ thanh cuộn: input scrollY 0→100 map opacity header 1→0, hoặc đổi màu (lúc đó có thể cần useNativeDriver: false cho màu).

const headerOpacity = scrollY.interpolate({
  inputRange: [0, 80],
  outputRange: [1, 0],
  extrapolate: "clamp",
});

8. Chạy nối tiếp / song song — sequence, parallel, stagger

Animated.sequence([
  Animated.timing(a, { toValue: 1, duration: 200, useNativeDriver: true }),
  Animated.parallel([
    Animated.timing(b, { toValue: 20, duration: 200, useNativeDriver: true }),
    Animated.spring(c, { toValue: 1, useNativeDriver: true }),
  ]),
]).start();

9. LayoutAnimation (đổi layout một lần — list expand)

Trước khi setState làm đổi kích thước subtree, có thể gọi LayoutAnimation.configureNext(...) (iOS mặc định, Android có thể cần bật UIManager.setLayoutAnimationEnabledExperimental trên bản cũ).

Không thay được toàn bộ Animated; dùng khi “ ô item mở rộng / thu nhỏ ” đơn giản.

10. Bước tiếp: Reanimated (+ Gesture Handler)

Expo và React Navigation thường kéo react-native-reanimated — chạy worklet trên native, chủ động hơn cho gesture phức tạp, shared transitions. Nên học đủ Animated cơ bản trước (timing, spring, transform, interpolate) rồi chuyển pattern sang Reanimated theo docs bản SDK bạn đang dùng.

Dữ liệu & gọi API — từ fetch đến token

Backend REST thường trả JSON. Trên RN bạn không có CORS như browser khi chạy native — vấn đề hay gặp là URL sai, HTTPS, timeout, và 401 / hết phiên.

Luồng chuẩn trong một màn hình

  1. State tối thiểu: data, loading, error (có thể gộp vào reducer sau).
  2. useEffect: khi mount (hoặc khi deps đổi) gọi hàm load().
  3. Kết xử: render 3 nhánh — loading / lỗi / có dữ liệu (kể cả mảng rỗng).

Mẫu fetch + JWT (rút gọn)

import { useEffect, useState } from "react";

const BASE = "https://your-api.example.com";

async function apiGet(path, token) {
  const res = await fetch(`${BASE}${path}`, {
    headers: {
      Accept: "application/json",
      ...(token ? { Authorization: `Bearer ${token}` } : {}),
    },
  });
  const text = await res.text();
  const body = text ? JSON.parse(text) : null;
  if (!res.ok) throw new Error(body?.message ?? `HTTP ${res.status}`);
  return body;
}

/** Custom hook ví dụ: load danh ví khi có token */
function useWallets(token) {
  const [state, setState] = useState({ loading: true, error: "", rows: [] });

  useEffect(() => {
    let cancelled = false;
    (async () => {
      try {
        setState((s) => ({ ...s, loading: true, error: "" }));
        const rows = await apiGet("/api/wallets", token);
        if (!cancelled) setState({ loading: false, error: "", rows });
      } catch (e) {
        if (!cancelled) setState((s) => ({ ...s, loading: false, error: String(e.message) }));
      }
    })();
    return () => { cancelled = true; }; // tránh setState sau unmount
  }, [token]);

  return state;
}

Axios — cùng mục này, ngay sau fetch

Axios không nằm mục riêng trên roadmap: nó thay thế / song song với fetch trong bước Hero — gọi API. Cài: npx expo install axios (hoặc npm i axios). Khác fetch: tự parse JSON, ném lỗi theo status (tuỳ cấu hình), baseURL, timeout, và đặc biệt interceptor (gắn token mọi request, xử lý 401 một chỗ). 

import axios from "axios";

const api = axios.create({
  baseURL: "https://your-api.example.com",
  timeout: 15_000,
  headers: { Accept: "application/json" },
});

api.interceptors.request.use((config) => {
  const token = globalThis.__accessToken; // thực tế: đọc từ context / memory sau khi load từ AsyncStorage
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

api.interceptors.response.use(
  (res) => res,
  (err) => {
    if (err.response?.status === 401) {
      /* logout hoặc refresh token */
    }
    return Promise.reject(err);
  }
);

export async function getWallets() {
  const { data } = await api.get("/api/wallets");
  return data;
}

export async function postTransaction(body) {
  const { data } = await api.post("/api/transactions", body);
  return data;
}

Học thứ tự gợi ý: nắm fetch + custom hook trước → chuyển sang axios khi muốn gom cấu hình và interceptor; TanStack Query phía dưới có thể bọc bất kỳ hàm async nào (fetch hoặc axios).

Lưu token & gọi lại sau login

  • @react-native-async-storage/async-storage (hoặc secure store tuỳ yêu cầu bảo mật).
  • Sau login: setItem('token', accessToken); khi khởi động app đọc token rồi quyết định vào Login hay Home.
  • 401 toàn cục: interceptor (nếu dùng axios) hoặc bọc apiGet để logout / refresh token một lần.

Gợi ý kiểm thử tay
